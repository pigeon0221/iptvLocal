## What's New

- **Fixed live channels restarting every few minutes in Jellyfin (and other tuners).** A channel could visibly drop and restart at regular intervals even when the stream was perfectly healthy. It now plays straight through without the interruption.
- **Fixed the dashboard's channel preview losing audio/video sync.** The built-in preview player could slip out of sync at regular intervals while playing; it now reconnects cleanly and stays in sync.
- **Update straight from the dashboard.** When a newer version is available, an **Update now** button now appears on the Dashboard and in **Settings → Server**. It downloads the new version and restarts for you — your data is kept and there's no command line needed. This needs the `rebeliptv/iptv:latest` image with the Docker socket mounted (see *Updating → In-app updates* below); version-pinned installs keep updating the usual way with `docker pull` + `docker compose up -d`.

## Docker

```bash
docker pull rebeliptv/iptv:1.3.0
```

### Docker Compose

```yaml
services:
  iptv:
    image: rebeliptv/iptv:1.3.0
    hostname: iptv
    container_name: iptv
    ports:
      - "8080:8080"
    # Environment variables are OPTIONAL — defaults work out of the box.
    # Uncomment any you need:
    # environment:
    #   TZ: "America/New_York"   # set to YOUR timezone, e.g. Europe/London, Etc/UTC
    volumes:
      - iptv-data:/app/data
    dns:
      - 8.8.8.8
      - 4.2.2.2
      - 1.1.1.1
    stop_grace_period: 30s
    restart: unless-stopped

volumes:
  iptv-data:
```

### Docker Run

```bash
docker run -d \
  --name iptv \
  -p 8080:8080 \
  -v iptv-data:/app/data \
  --dns 8.8.8.8 \
  --dns 4.2.2.2 \
  --dns 1.1.1.1 \
  --restart unless-stopped \
  rebeliptv/iptv:1.3.0

# Environment variables are optional (defaults work). Add -e flags as needed, e.g.:
#   -e TZ=America/New_York   (set to YOUR timezone — e.g. Europe/London, Etc/UTC)
```

### Updating

```bash
docker pull rebeliptv/iptv:1.3.0
docker compose up -d
```

#### In-app updates (optional)

To upgrade from the dashboard's **Update now** button instead of the
command line, run the published `:latest` image and mount the Docker
socket so the container can update itself:

```yaml
services:
  iptv:
    image: rebeliptv/iptv:latest
    hostname: iptv
    container_name: iptv
    ports:
      - "8080:8080"
    # Environment variables are OPTIONAL — defaults work out of the box.
    # Uncomment any you need:
    # environment:
    #   TZ: "America/New_York"   # set to YOUR timezone, e.g. Europe/London, Etc/UTC
    volumes:
      - iptv-data:/app/data
      - /var/run/docker.sock:/var/run/docker.sock
    dns:
      - 8.8.8.8
      - 4.2.2.2
      - 1.1.1.1
    stop_grace_period: 30s
    restart: unless-stopped

volumes:
  iptv-data:
```

> Mounting the Docker socket gives the container control of the Docker
> daemon on the host — enable it only if you're comfortable with that.
> The button appears only on the `:latest` image when a newer release
> exists; it's hidden on version-pinned images.

## Endpoints

| URL | Description |
|-----|-------------|
| `http://<server>:8080/playlist?key=YOUR_KEY` | M3U playlist |
| `http://<server>:8080/epg?key=YOUR_KEY` | EPG guide (XML) |
| `http://<server>:8080/epg.gz?key=YOUR_KEY` | EPG guide (gzip) |
| `http://<server>:8080` | Web dashboard |

> **Note:** The `?key=` parameter is only required if an API key has been generated in the web dashboard under **Settings**.
