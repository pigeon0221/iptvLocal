## What's New

- Restored playback for channels whose upstream retired their old streaming infrastructure — scraper and proxy were rewritten end-to-end against the new player chain (channels resolve on demand, segments stream with per-channel auth)
- Multiple viewers of the same channel now share a single upstream fetch — the second viewer joins the in-flight download and gets the bytes from memory instead of triggering a duplicate fetch upstream
- Significantly less buffering on slower upstreams — proxy pre-warms the live edge as soon as the playlist is fetched, streams bytes byte-for-byte to the player as they arrive (no full-segment buffering before first byte), and tees the stream into a 30-second cache so subsequent requests hit memory
- AES-128 encrypted streams now decrypt and play correctly across all affected channels
- Sports events drop cleanly at midnight — at 12:00am the day after, all of yesterday's events are removed except those still live within their post-game buffer, no separate finished-events file persisted on disk
- Web UI play-time display no longer runs away on long-running live channels — was occasionally jumping by hundreds of hours after a buffer hiccup; the seekbar now behaves like a live ticker and stays sane regardless of how long a channel has been playing
- Web UI player auto-recovers when playback is genuinely frozen — a minimal safety net hard-snaps back to the live edge only when the player has been stuck for ~90 seconds, replacing an earlier version that turned every transient hiccup into a visible rubber-banding seek
- Default Docker image is ~180 MB smaller (chromium / nss / freetype / harfbuzz removed; no longer required by the new auth flow)

## Docker

```bash
docker pull rebeliptv/iptv:1.1.6
```

### Docker Compose

```yaml
services:
  iptv:
    image: rebeliptv/iptv:1.1.6
    hostname: iptv
    container_name: iptv
    ports:
      - "8080:8080"
    environment:
      TZ: "America/New_York"
    volumes:
      - ./data:/app/data
    stop_grace_period: 30s
    restart: unless-stopped
```

### Docker Run

```bash
docker run -d \
  --name iptv \
  -p 8080:8080 \
  -e TZ=America/New_York \
  -v ./data:/app/data \
  --restart unless-stopped \
  rebeliptv/iptv:1.1.6
```

### Updating

```bash
docker pull rebeliptv/iptv:1.1.6
docker compose up -d
```

## Endpoints

| URL | Description |
|-----|-------------|
| `http://<server>:8080/playlist?key=YOUR_KEY` | M3U playlist |
| `http://<server>:8080/epg?key=YOUR_KEY` | EPG guide (XML) |
| `http://<server>:8080/epg.gz?key=YOUR_KEY` | EPG guide (gzip) |
| `http://<server>:8080` | Web dashboard |

> **Note:** The `?key=` parameter is only required if an API key has been generated in the web dashboard under **Settings**.
