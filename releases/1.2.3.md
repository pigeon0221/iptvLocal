## What's New

- Games that go to overtime / extra innings stay watchable — a live event now stays in the playlist and the guide (with its guide block extended to cover the overtime) until it actually ends, instead of disappearing or flipping to "Postgame" when its scheduled time runs out
- Fixed live streams dropping when the guide refreshed — the program guide (XMLTV) is now generated once and briefly cached, so a TV app rapidly polling the guide (Jellyfin fires it many times in a few seconds) no longer pegs the server and interrupts everyone's active streams
- New `USE_PROXY` option for regions where providers are blocked — set `USE_PROXY=true` in your environment and the app routes all source/provider connections through Rebel IPTV's proxy (which resolves and connects from a region where those sources aren't restricted), so an instance behind a country block or a poisoned DNS can still reach them. Off by default; only source traffic is proxied — the dashboard is unaffected
- Fixed live events not playing on TV / M3U-tuner clients (e.g. Jellyfin, on both the TV app and web) — the continuous stream now resolves an event's source the same way the web player does, so events start instead of failing silently. A source that can't be played now returns a clear error rather than an empty stream that makes a tuner retry forever
- Clear upgrade prompt when an instance is too old — when a build is no longer supported and can't start up, the container log now shows a plain-English "please upgrade" message instead of silently retrying forever
- Fixed the live viewer count and watch time not being recorded for tuner (continuous-stream) sessions
- Fixed connection failures from attempting providers over IPv6 — the app now connects over IPv4 only, matching the container's network

## Docker

```bash
docker pull rebeliptv/iptv:1.2.3
```

### Docker Compose

```yaml
services:
  iptv:
    image: rebeliptv/iptv:1.2.3
    hostname: iptv
    container_name: iptv
    ports:
      - "8080:8080"
    environment:
      TZ: "America/New_York"
    volumes:
      - iptv-data:/app/data
    dns:
      - 8.8.8.8
      - 4.2.2.2
      - 1.1.1.1
    stop_grace_period: 30s
    restart: unless-stopped

volumes:
  iptv-data:
```

### Docker Run

```bash
docker run -d \
  --name iptv \
  -p 8080:8080 \
  -e TZ=America/New_York \
  -v iptv-data:/app/data \
  --dns 8.8.8.8 \
  --dns 4.2.2.2 \
  --dns 1.1.1.1 \
  --restart unless-stopped \
  rebeliptv/iptv:1.2.3
```

### Updating

```bash
docker pull rebeliptv/iptv:1.2.3
docker compose up -d
```

## Endpoints

| URL | Description |
|-----|-------------|
| `http://<server>:8080/playlist?key=YOUR_KEY` | M3U playlist |
| `http://<server>:8080/epg?key=YOUR_KEY` | EPG guide (XML) |
| `http://<server>:8080/epg.gz?key=YOUR_KEY` | EPG guide (gzip) |
| `http://<server>:8080` | Web dashboard |

> **Note:** The `?key=` parameter is only required if an API key has been generated in the web dashboard under **Settings**.
