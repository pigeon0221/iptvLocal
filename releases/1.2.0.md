## What's New

- Rebuilt the core engine for speed and stability — noticeably faster startup, lower memory use, and steadier long-running playback
- Significantly smaller, more secure container image — far fewer system dependencies, which means a smaller attack surface and quicker pulls/updates
- Sports event artwork is now generated inside the app — clean "away vs home" matchup logos with the league crest render for both the channel grid and the EPG guide, with no external image tools required
- Your setup carries over automatically — existing custom M3U sources and your generated API key are migrated on the first launch of this version, so your saved playlist/EPG URLs (including the `?key=…` ones) keep working with nothing to re-add or re-copy

## Docker

```bash
docker pull rebeliptv/iptv:1.2.0
```

### Docker Compose

```yaml
services:
  iptv:
    image: rebeliptv/iptv:1.2.0
    hostname: iptv
    container_name: iptv
    ports:
      - "8080:8080"
    environment:
      TZ: "America/New_York"
    volumes:
      - iptv-data:/app/data
    dns:
      - 8.8.8.8
      - 4.2.2.2
      - 1.1.1.1
    stop_grace_period: 30s
    restart: unless-stopped

volumes:
  iptv-data:
```

### Docker Run

```bash
docker run -d \
  --name iptv \
  -p 8080:8080 \
  -e TZ=America/New_York \
  -v iptv-data:/app/data \
  --dns 8.8.8.8 \
  --dns 4.2.2.2 \
  --dns 1.1.1.1 \
  --restart unless-stopped \
  rebeliptv/iptv:1.2.0
```

### Updating

```bash
docker pull rebeliptv/iptv:1.2.0
docker compose up -d
```

## Endpoints

| URL | Description |
|-----|-------------|
| `http://<server>:8080/playlist?key=YOUR_KEY` | M3U playlist |
| `http://<server>:8080/epg?key=YOUR_KEY` | EPG guide (XML) |
| `http://<server>:8080/epg.gz?key=YOUR_KEY` | EPG guide (gzip) |
| `http://<server>:8080` | Web dashboard |

> **Note:** The `?key=` parameter is only required if an API key has been generated in the web dashboard under **Settings**.
