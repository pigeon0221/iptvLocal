## What's New

- Fixed live TV streams (especially on M3U-tuner clients like Jellyfin) still occasionally cutting out or switching to a backup feed during a background guide refresh — even when the feed you were watching was perfectly healthy
- Fixed DNS resolution for custom sources — they now resolve on your own local network, so a custom source pointing at a private or LAN address (for example a home media server or another box on your network) loads correctly
- Custom sources now get a TV guide — if your custom playlist links an EPG (the `url-tvg`/`x-tvg-url` line most IPTV playlists include, including `.gz` guides), its programme listings now appear in the guide and "now playing" for those channels

## Docker

```bash
docker pull rebeliptv/iptv:1.2.5
```

### Docker Compose

```yaml
services:
  iptv:
    image: rebeliptv/iptv:1.2.5
    hostname: iptv
    container_name: iptv
    ports:
      - "8080:8080"
    # Environment variables are OPTIONAL — defaults work out of the box.
    # Uncomment any you need:
    # environment:
    #   TZ: "America/New_York"   # set to YOUR timezone, e.g. Europe/London, Etc/UTC
    #   USE_PROXY: "true"        # only if providers are blocked in your region
    volumes:
      - iptv-data:/app/data
    dns:
      - 8.8.8.8
      - 4.2.2.2
      - 1.1.1.1
    stop_grace_period: 30s
    restart: unless-stopped

volumes:
  iptv-data:
```

### Docker Run

```bash
docker run -d \
  --name iptv \
  -p 8080:8080 \
  -v iptv-data:/app/data \
  --dns 8.8.8.8 \
  --dns 4.2.2.2 \
  --dns 1.1.1.1 \
  --restart unless-stopped \
  rebeliptv/iptv:1.2.5

# Environment variables are optional (defaults work). Add -e flags as needed, e.g.:
#   -e TZ=America/New_York   (set to YOUR timezone — e.g. Europe/London, Etc/UTC)
#   -e USE_PROXY=true        (only if providers are blocked in your region)
```

### Updating

```bash
docker pull rebeliptv/iptv:1.2.5
docker compose up -d
```

## Endpoints

| URL | Description |
|-----|-------------|
| `http://<server>:8080/playlist?key=YOUR_KEY` | M3U playlist |
| `http://<server>:8080/epg?key=YOUR_KEY` | EPG guide (XML) |
| `http://<server>:8080/epg.gz?key=YOUR_KEY` | EPG guide (gzip) |
| `http://<server>:8080` | Web dashboard |

> **Note:** The `?key=` parameter is only required if an API key has been generated in the web dashboard under **Settings**.
